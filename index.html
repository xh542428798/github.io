绑定和非绑定实体

  • 属于该项目的所有实体都会自动保存在 Reference Person Keyword Project.Save()
  • 实体始终分配给项目，但可以是“绑定”或“未绑定”。
  • “绑定”表示该实体是相应项目集合的元素： -> , Person Project.Persons Reference -> Project.References
  • 每个实体都拥有属性。 IsBound
  • 这种方法使我们能够自由地创建从未保存的实体。

Citavi插件开发VS配置

创建一个新的 Citavi 3 Add-On Development 文件夹

Citavi 程序文件夹通常位于 Windows Vista 或 Windows 7 中的 Program Files 文件夹下方。这意味着它受到 Windows UAC 功能的保护。这可能会使附加开发变得复杂。因此，作为第一步，在不受 UAC 保护的位置创建程序文件夹的副本是有意义的，一般放DEFG等盘里。

  1. 转到我的文档文件夹。
  2. 新建一个文件夹。
  3. 将文件夹命名为（例如）Citavi 3 Add-On Development。

复制程序文件

  1. 转到 Program Files\Citavi 3 文件夹
  2. 按 Ctrl + A 或选择所有文件夹。
  3. 将所有选定的文件夹复制到剪贴板。

插入程序文件

  1. 转到之前创建的 Citavi 3 Add-On Development 文件夹
  2. 粘贴从剪贴板复制的文件夹。

创建附加组件文件夹

  1. 在 Citavi 3 Add-On Development 文件夹中创建一个新文件夹。
  2. 将此文件夹命名为 AddOns。

创建新项目，每个新项目都需要操作

  1. 启动 Visual Studio。
  2. 单击“新建项目”。

选择项目类型和名称

  1. 从下拉列表中选择 .NET Framework 4.6.1 (Citavi 6的框架是4.6.1)
  2. 选择类库（Class library）。
  3. 为附加组件输入一个有意义的名称，在我们的示例 ShortTitleFilterDemoAddOn 中。
  4. 单击“确定”。
